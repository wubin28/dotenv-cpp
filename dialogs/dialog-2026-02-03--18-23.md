## Sider with claude sonnet 4.5:

```
根据你的输出，问题是 CMake 没有被添加到系统的 PATH 环境变量中。Visual Studio 安装的 CMake 默认不会自动添加到 PATH。

## 解决方案

### 方案 1：将 Visual Studio 的 CMake 添加到 PATH (亲测有效)

Visual Studio 2022 的 CMake 通常位于：
```
C:\Program Files\Microsoft Visual Studio\2022\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin
```

对于你的 Visual Studio 2026（版本18），路径可能是：
```
C:\Program Files\Microsoft Visual Studio\18\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin
```

**添加到 PATH 的步骤：**

1. 在 PowerShell 中运行以下命令验证路径是否存在：
```powershell
Test-Path "C:\Program Files\Microsoft Visual Studio\18\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin"
```

2. 如果路径存在，将其添加到用户环境变量：
```powershell
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Program Files\Microsoft Visual Studio\18\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin", "User")
```

3. 关闭并重新打开 PowerShell，然后测试：
```powershell
cmake --version
```

### 方案 2：使用 Visual Studio 的开发者命令提示符

直接使用 Visual Studio Developer PowerShell，它会自动配置所有工具的路径：
- 在开始菜单搜索 "Developer PowerShell for VS 2026"
- 在里面运行 `cmake --version` 应该就能工作

### 方案 3：独立安装 CMake

从 [cmake.org](https://cmake.org/download/) 下载独立安装包，安装时选择 "Add CMake to the system PATH"。
