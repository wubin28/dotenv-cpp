# Understand the Code Skill

Help a new developer quickly understand a C++ CMake project by researching it and producing a structured analysis document.

## Workflow

### 1. Read Key Project Files

Read these files in order, stopping when you have enough answers for all three questions:

1. `README.md` — overview, features, usage, installation
2. `CMakeLists.txt` — build targets, test setup, dependencies
3. Header files (e.g., `include/**/*.h`) — API surface, key types
4. Test files (e.g., `tests/*.cc`, `tests/*.cpp`) — how tests are organized and what fixtures they use
5. Example source files (e.g., `example.cpp`) — how the library is used end-to-end
6. Build directory if it exists (e.g., `build/CMakeCache.txt`) — already-configured generators, options

### 2. Answer the Three Questions

Structure your answer around these three questions:

**Q1 — Project overview** (功能、特点、优势、劣势、适用场景):
- Main functionality in one paragraph
- Key technical characteristics (header-only? dependencies? OS support?)
- Advantages and disadvantages
- Recommended use cases

**Q2 — Build and run on Windows 11 + VS2022 + PowerShell**:
- Prerequisites (CMake version, Visual Studio workload, etc.)
- Exact PowerShell commands to configure and build, with flags like `-DBUILD_TESTS=ON` only when needed
- How to create any required data files (e.g., `.env` or `.env.example`) with realistic sample content
- How to run the compiled binary and interpret its output

**Q3 — Run automated tests**:
- How to run tests via CTest or the test runner binary, with exact commands
- What to expect in the output when tests pass

### 3. Output Guidelines

- Be specific enough that the reader can copy-paste each command without guessing.
- Mention any files that must exist before running (e.g., `.env.example` for dotenv-cpp).
- If a build directory already exists, provide commands for both "fresh build" and "incremental build" scenarios.
- Warn about common Windows-specific pitfalls (path separators, `_putenv_s` vs `setenv`, Debug vs Release configs).

### 4. Save as Timestamped File

After writing the answer, save it to:

```
dialogs/understand-the-code-<timestamp>.md
```

where `<timestamp>` follows the format `YYYY-MM-DD--HH-MM` (e.g., `2026-03-06--20-15`), using the current local time. Create the `dialogs/` directory if it does not exist.

The file must include the original three questions as headings followed by your answers, formatted in Markdown.
