## 关于这个测试与 dotenv-cpp 的关系

**结论：这个测试本身完全不使用 dotenv-cpp 库。**

### 测试实际做了什么

```cpp
TEST(JoinWithInventory, AC_H1_JoinKeepsOnlySalesWithMatchingProduct) {
    auto sales     = make_sales();      // 用硬编码字符串 kSalesCsv 构造数据
    auto inventory = make_inventory();  // 用硬编码字符串 kInventoryCsv 构造数据

    auto joined = join_with_inventory(sales, inventory);
    ASSERT_EQ(joined.size(), 4u);
}
```

helper `make_sales()` / `make_inventory()` 直接调用 `parse_csv_content()` 处理内存字符串，**完全绕开了文件读取和配置加载**。没有任何一层涉及 .env 或 dotenv-cpp。

### dotenv-cpp 在哪里被用到

dotenv-cpp 只在 config.cpp 里登场：

```cpp
dotenv::init(env_file.c_str());
const auto sales_file = dotenv::getenv("SALES_FILE", "");
// ...
```

它被 `load_config()` 调用，而 `load_config()` 只在 main.cpp 的生产入口中使用。测试文件根本不包含 config.h，因此对 dotenv-cpp 零依赖。

---

### AC-H1 对应的 env 定义在哪里

AC-H1 的环境变量规格定义在 user-story-csv-reporter.md：

```
SALES_FILE=csv-reporter/data/sales.csv
INVENTORY_FILE=csv-reporter/data/inventory.csv
DELIMITER=,
MIN_AMOUNT=1000
OUTPUT_FORMAT=json
```

这是**给生产程序看的运行时配置**，需要用户在项目根目录手动创建 .env 文件。当前 workspace 里没有 .env 文件——测试之所以能独立运行，正是因为测试代码里用硬编码数据模拟了 AC-H1 场景（4 条销售记录、threshold=1000），完全不依赖 .env。

---

**总结：** dotenv-cpp 是"生产程序从 .env 读配置"的基础设施；这个单元测试绕过了整个配置层，直接验证业务逻辑函数的正确性，是典型的纯单元测试写法。
